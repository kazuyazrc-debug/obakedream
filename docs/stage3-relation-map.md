# Stage3 Relation Map

40モチーフ時点の重点レビュー群を整理します。全組み合わせを網羅するのではなく、次の20件追加前に誤検出や意味重複が起きやすい群を確認します。

## 水系

対象: `water / sea / rain / river`

- `sea` は大きな感情や人生の潮目
- `river` は時間の流れや移行
- `rain` は感情の放出や浄化
- `water` は基礎象徴

方針: 具体motifが明確なら具体側を優先し、`water` は補助象徴として扱う。

## 場所・境界

対象: `house / childhood_home / key / toilet / mirror`

- `childhood_home` は `house` より具体
- `key` は境界を開閉する手がかり
- `toilet` は手放しと浄化
- `mirror` は自己認識

方針: 家系は具体性で整理し、鍵/トイレ/鏡は家の中の機能としてneighborに留める。

## 評価・責任

対象: `school / exam / workplace / being_late`

- `exam` は評価イベント
- `school` は学びや評価環境
- `workplace` は社会的役割
- `being_late` は期限不安

方針: イベントが明確なら `exam` を優先。遅刻は環境motifと併存させる。

## 移動・進路

対象: `train / car / elevator / road / bridge / stairs`

- `road` は進路と選択
- `bridge` は移行と接続
- `stairs` は段階変化
- `elevator` は急な上下変化
- `train` は集団の流れ
- `car` は自分で進む力

方針: 乗り物は移動主体の違い、場所motifは進路の質で整理する。

## 人物・関係

対象: `friend / family / stranger / ex_partner`

- `friend` は対等な関係
- `family` は近い役割と責任
- `stranger` は未知の自己像
- `ex_partner` は過去の関係と未整理感

方針: 元恋人は恋愛派生を広げすぎず、過去関係の整理として扱う。

## 不安・防衛

対象: `being_chased / fighting / falling / fire`

- `being_chased` は回避とプレッシャー
- `fighting` は防衛と主張
- `falling` は足場不安
- `fire` は勢いと感情の熱

方針: 行動が明確な場合は行動motifを優先し、火は感情エネルギーとしてneighborにする。

## 回復・始まり

対象: `baby / hospital / water / rain`

- `baby` は新しい可能性
- `hospital` は点検と回復
- `water` と `rain` は浄化

方針: 体調やケアが明確なら `hospital`、始まりが明確なら `baby` を中心にする。
