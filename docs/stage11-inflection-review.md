# Stage11 行動系活用形レビュー

Stage11では、Stage10までに phrase 追加で救った活用形と、未対応として残すものを文書化します。batch-07に行動系motifを追加する場合の設計前提として使います。

|---|---|---|---|

## 未対応一覧（Stage11時点）

以下は「phrase追加で救えるが今は見送る」と「設計として拾わない」に分類します。

### phrase追加候補（次回検討）

| 表現 | 対象motif | 追加しない理由 / 追加する場合の条件 |
|---|---|---|
| `なくなっていた`（状態変化） | `losing` | なくなる（状態変化）は失くす（能動的紛失）と意味が異なる。phraseを追加するなら `losing` の意味定義を変えずに別phraseとして追加できるか確認が必要 |
| `走り出した`・`走り去った` | `running` | 走るの後続形として拾えるが、逃げる・去るの文脈が強くなりすぎる可能性あり。batch-07の行動系追加前に再検討 |
| `どこを探しても見つからない` | `searching` | 既存phrase「探しても見つからない」で部分一致で救えるか要確認。実質的に対応済みの可能性あり |

### 未対応として残す（設計方針）

| 表現 | 対象motif | 理由 |
|---|---|---|
| `気持ちだけが先走る` | `running` | 比喩表現。辞書肥大を避け、物理的な走る行動に限定する |
| `心を洗い流したい` | `cleaning` | 比喩的浄化表現。掃除するmotifへ寄せすぎない方針 |
| `ど忘れしていた` | `forgetting` | 口語的。標準的な言い換えではないため辞書に入れない |
| `扉が開きっぱなし` | `opening` | 「開けっぱなし」という状態表現。開ける行動ではないため別設計が必要 |
| `なくなっていた`（状態） | `losing` | Stage11 audit fixture で未対応を確認済み（`stage11-unhandled-losing-state-expression`） |
| `消えた記憶を追いかける` | `searching` | 抽象的な探索表現。具体的な探し物文脈に限定する方針（`stage11-unhandled-searching-extended`） |
| `どこだか分からなくなって` | `lost` | 迂回表現。lostのphrase/alias設計範囲外（`stage11-lost-paraphrase-unhandled`） |

## batch-07設計への示唆

行動系motifを batch-07 に追加する場合は、以下の設計前提を参照してください。

### phraseで救う対象の優先順位

1. 夢文として最も自然な活用形（て形、た形）を最低1件
2. 代表的なモチーフ名詞 + 行動の組み合わせ（財布を失くして、ドアを開いた等）
3. 口語・比喩・状態表現は原則として未対応に残す

### 活用形追加の安全条件

- phrase追加は既存motifへの影響（over-extraction）を audit fixture で事前確認する
- 短い語のphrase化（2文字以下）は warning-fix-soon になるため避ける
- 意味が微妙に異なる活用形（なくす vs なくなる）は別motifまたは未対応として扱う

### batch-07で行動系を追加する場合の注意

- `笑う`、`泣く`、`謝る`、`隠れる` 等の感情行動は、Stage11未対応一覧で同様のパターンが出ないか確認してから追加する
- 行動系の活用形が既存motifの alias に含まれていないかを validator で確認してから phrase 設計する
